# OfferPilot / 求职作战台

本地优先的求职作战台。面向 Boss 直聘求职场景的本地 AI 求职手账。

它不接 API，不做自动投递，不做爬虫，不做后端。它帮助求职者把“看岗位、生成分析 Prompt、承接外部 AI 结果、提取 Boss 打招呼话术、保存岗位台账、跟踪沟通状态”整理成一条稳定流程。

## 项目定位

OfferPilot v0.1 不是 AI 自动分析器，而是：

> 本地优先的 AI 求职工作流容器。

真正的模型推理由用户自带的外部 AI（ChatGPT / Claude / Gemini）完成，工具只负责输入标准化、Prompt 生成、结果承接与台账沉淀。

## 当前模式：Manual Mode（不接 AI API）

v0.1 采用 Manual Mode：

- 工具本身**不接入** OpenAI / Claude / Gemini 等任何 AI API，也不做 BYOK、不做后端。
- 分析能力来自你自己选择的外部 AI：工具生成 Prompt，你复制到外部 AI，再把结果贴回工具。
- 因此工具不会“自动分析”，它是一个把求职动作标准化的**整理工具**。

## 核心闭环

```
简历/偏好配置  →  录入岗位 JD  →  生成分析 Prompt  →  复制到外部 AI
      →  把 AI 结果粘贴回岗位  →  沉淀报告原文 / 编辑 Boss 话术
      →  维护沟通状态  →  面试前从列表回看完整岗位
```

## 本地运行

要求 Node 18+（开发环境使用 Node 24）。

```bash
# 安装依赖
npm install

# 启动开发服务器（默认 http://localhost:5173）
npm run dev

# TypeScript 类型检查（strict，期望 0 error）
npm run typecheck

# 存储层自测（期望 30/30 通过）
npm run selftest

# 生产构建
npm run build
```

所有数据保存在浏览器 localStorage，本地优先、刷新不丢，无需登录、无需后端。

## v0.1.0 功能清单

1. 简历 / 偏好全局配置（9 字段，覆盖式保存，刷新回显）
2. 岗位台账列表（空状态、列表展示、新建入口、点击进入详情）
3. 岗位主战场基础信息录入与保存（新建创建 / 编辑更新）
4. 结构化分析 Prompt 生成 + 一键复制（含外部 AI 操作提示）
5. 外部 AI 结果原文兜底承接（原文必存、粘贴时间、解析状态标记）
6. 分析报告原文展示 + Boss 打招呼话术编辑 / 保存 / 复制
7. 沟通状态流转（未沟通 / 已打招呼 / 已回复 / 已约面 / 已拒绝 / 已结束，即时持久化）
8. 列表 ↔ 详情回看闭环（面试前可回溯报告与话术）
9. 全本地存储（localStorage），刷新不丢

## 当前能力边界（v0.1 不做）

1. 不接 OpenAI / Claude / Gemini API
2. 不做 BYOK
3. 不做后端
4. 不做登录注册
5. 不做 Boss 自动投递
6. 不做 Boss 爬虫
7. 不做浏览器插件
8. 不做 PDF / Word 简历解析
9. 不做云端同步
10. 不做复杂数据统计
11. 不做完整 AI Chat
12. 不自动结构化解析 AI 报告（v0.1 只兜底保存原文 + 手改话术）
