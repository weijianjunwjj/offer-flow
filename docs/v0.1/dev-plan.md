# OfferFlow / Offer来了 v0.1 开发计划

## 总原则

只做 P0。

不接 API，不做 BYOK，不做后端，不做爬虫，不做 PDF / Word 解析，不做登录，不做云同步，不做复杂统计。

开发顺序：

> 先骨架后血肉，先存后取，先兜底后结构。

## Step 0：数据与本地存储底座

目标：

搭好全局配置和岗位记录的最小本地读写能力。

范围：

1. 全局配置保存 / 读取 / 更新 / 清除
2. 岗位记录创建 / 读取 / 更新 / 删除 / 列表
3. 支持浏览器存储
4. 支持测试用内存存储
5. 自测覆盖最小 CRUD

不做：

1. 不做页面
2. 不做 Prompt
3. 不做 AI 结果承接
4. 不做报告
5. 不做 Boss 话术

## Step 1：简历 / 偏好配置页

目标：

做出全局配置表单，保存后刷新仍在。

范围：

1. 简历文本
2. 项目经历
3. 目标城市
4. 目标岗位方向
5. 期望薪资
6. 是否接受外包
7. 是否接受加班
8. 当前求职重点
9. 个人短板说明
10. 保存与回显

## Step 2：岗位台账列表空壳

目标：

打通列表主干道。

范围：

1. 无数据空状态
2. 有数据列表展示
3. 新建岗位入口
4. 点击列表项进入岗位主战场回看态

## Step 3：岗位主战场基础信息 + 保存

目标：

打通“创建一条岗位记录”的最小闭环。

范围：

1. 公司名
2. 岗位名
3. 城市
4. 薪资范围
5. 岗位 JD 原文
6. 保存岗位
7. 回列表展示

## Step 4：Prompt 生成 + 复制

目标：

基于全局配置 + 岗位信息生成完整 Prompt。

范围：

1. 生成 Prompt
2. 展示 Prompt
3. 一键复制 Prompt
4. 外部 AI 使用引导文案
5. 检查无 `{{}}` 残留变量

## Step 5：AI 结果兜底承接

目标：

用户粘贴外部 AI 结果后，原文完整保存。

范围：

1. AI 结果粘贴框
2. 粘贴后保存完整原文
3. 保存粘贴时间
4. 标记解析状态
5. 解析失败也不阻断
6. 给用户“已保存原文”反馈

## Step 6：报告展示 + Boss 话术编辑

目标：

展示报告，提取或手动编辑 Boss 打招呼话术。

范围：

1. 报告展示：分节展示或原文展示
2. Boss 话术编辑框
3. 复制报告
4. 复制话术
5. 话术编辑后保存
6. 刷新后仍保留

优先级：

- P0：完整原文保存
- P0：Boss 话术提取 / 手动编辑
- P1：投递建议提取
- P1：岗位关键词提取
- P2：完整结构化解析

## Step 7：沟通状态流转

目标：

岗位列表和详情中可以修改沟通状态。

范围：

1. 未沟通
2. 已打招呼
3. 已回复
4. 已约面
5. 已拒绝
6. 已结束

要求：

1. 可从任意状态切换到任意状态
2. 不做强制顺序
3. 不做提醒
4. 不做自动推进
5. 状态持久化

## Step 8：回看闭环 + 文案收口

目标：

让用户面试前可以回溯岗位分析，同时避免误解为自动 AI 工具。

范围：

1. 列表项点击进入岗位详情
2. 回显基础信息
3. 回显 AI 原文
4. 回显报告
5. 回显 Boss 话术
6. 回显沟通状态
7. 补齐空状态文案
8. 补齐“去外部 AI 粘贴再贴回”的引导文案
9. 避免使用“自动 AI 分析”等误导表达
