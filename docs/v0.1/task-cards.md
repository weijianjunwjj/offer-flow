# OfferPilot / 求职作战台 v0.1 任务卡

## 任务纪律

每次只执行一张任务卡。  
完成后必须停下，输出改动文件、自测方式、自测结果、遗留风险。  
未确认前，不进入下一张任务卡。

---

## Task 0：初始化仓库 + 导入 Step 0

目标：

初始化本地仓库，建立 Vue 3 + Vite + TypeScript 项目，导入 Step 0 存储层。

范围：

1. 初始化项目
2. 建立 docs 目录
3. 写入产品文档、开发计划、任务卡、验收标准
4. 导入 Step 0 存储层到 src/storage
5. 放置 selftest
6. 跑通类型检查和自测

禁止：

1. 不做页面
2. 不做 Prompt
3. 不做 AI 结果承接
4. 不做报告
5. 不做 Boss 话术
6. 不接 API
7. 不做 BYOK
8. 不做后端

验收：

1. 仓库能启动基本项目
2. docs 文件存在
3. src/storage 文件存在
4. selftest 能跑
5. typecheck 通过
6. Step 0 存储层能保存 / 读取 / 更新 / 删除配置与岗位记录

## Task 5：AI 结果兜底承接

目标：

用户粘贴外部 AI 结果后，完整保存原文。

范围：

1. AI 结果粘贴框
2. 原文保存
3. 粘贴时间
4. 解析状态标记
5. 保存反馈

禁止：

1. 不做复杂结构化解析
2. 不因解析失败阻断
3. 不丢原文

验收：

1. 任意格式文本都能保存
2. 刷新后原文仍在
3. 解析失败不报错
4. 有已保存反馈
