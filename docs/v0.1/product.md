# OfferPilot / 求职作战台 v0.1 产品文档

## 1. 一句话定位

OfferPilot v0.1 是一个本地优先的 AI 求职手账，把 Boss 直聘的“乱投、乱聊、乱记”收束成一条有准备的求职流水线。

## 2. 它是什么

OfferPilot v0.1 是：

- 求职流程容器
- Boss 求职作战台
- Prompt 工作台
- 岗位分析台账
- 外部 AI 结果承接器

## 3. 它不是什么

OfferPilot v0.1 不是：

- AI 大脑
- 自动分析器
- 自动投递机器人
- 招聘 SaaS
- 简历编辑器
- Boss 爬虫
- 完整 AI Chat 产品

## 4. 核心逻辑

工具负责：

1. 输入标准化
2. Prompt 标准化
3. 外部 AI 结果承接
4. 岗位台账沉淀
5. Boss 话术提取
6. 沟通状态管理

外部 AI 负责：

1. 岗位分析
2. 匹配判断
3. 简历建议
4. 面试准备
5. Boss 打招呼话术生成

## 5. 目标用户

第一目标用户：

正在使用 Boss 直聘找工作的前端开发者。

典型画像：

1. 1-8 年经验
2. Vue / React / TypeScript 方向
3. 关注中后台、管理系统、小程序、uniapp、AI 前端岗位
4. 有项目经历，但不知道如何对齐岗位 JD
5. Boss 打招呼话术比较随意
6. 投递记录混乱
7. 面试准备缺乏结构
8. 求职时容易焦虑、低效、乱投

## 6. 核心价值

按优先级排序：

1. 把简历 / 项目 / 偏好沉淀成全局配置，避免重复填写
2. 一键生成结构稳定的分析 Prompt
3. 兜底承接外部 AI 返回结果，先保证不丢，再追求结构化
4. 抽出可直接用的 Boss 打招呼话术
5. 用台账管理岗位沟通状态
6. 面试前可以回溯对应岗位分析

## 7. v0.1 三个视图

### 7.1 简历 / 偏好配置

目标：

保存用户的全局求职基础信息，避免每次分析岗位都重复填写。

字段：

- 简历文本
- 项目经历
- 目标城市
- 目标岗位方向
- 期望薪资
- 是否接受外包
- 是否接受加班
- 当前求职重点：求稳 / 涨薪 / 履历 / 技术成长
- 个人短板说明：学历 / 年限 / 离职原因等

### 7.2 岗位主战场

目标：

完成一次岗位分析的完整动作。

流程：

1. 新建岗位
2. 输入公司名
3. 输入岗位名
4. 输入城市
5. 输入薪资范围
6. 输入 Boss 岗位 JD
7. 生成 Prompt
8. 一键复制 Prompt
9. 用户去 ChatGPT / Claude / Gemini 粘贴
10. 用户把外部 AI 返回结果粘贴回工具
11. 工具保存完整原文
12. 工具展示报告
13. 工具提取 / 编辑 Boss 打招呼话术
14. 用户复制话术
15. 用户保存岗位记录

### 7.3 岗位台账列表

目标：

统一管理所有已分析岗位。

列表字段：

- 公司名
- 岗位名
- 城市
- 薪资范围
- 岗位类型
- 匹配度
- 投递建议
- 沟通状态
- 创建时间
- 最近更新时间

沟通状态：

- 未沟通
- 已打招呼
- 已回复
- 已约面
- 已拒绝
- 已结束

## 8. Prompt 输出要求

外部 AI 返回结果应包含：

1. 岗位类型判断
2. 岗位关键词
3. 技术栈匹配度
4. 项目经验匹配度
5. 优势命中
6. 风险短板
7. 简历优化建议
8. 面试准备清单
9. 投递建议
10. Boss 打招呼话术

投递建议四档：

1. 强烈建议沟通
2. 可以沟通
3. 谨慎沟通
4. 不建议浪费精力

## 9. AI 结果承接原则

v0.1 的铁律：

> 原文必存，结构化尽力，话术可手改。

要求：

1. 粘贴 AI 结果后，完整原文必须保存
2. 解析失败不能阻断保存
3. 至少允许用户手动编辑 Boss 打招呼话术
4. 至少允许用户查看完整报告原文
5. 不追求 v0.1 自动精准拆十个字段
6. 不因为解析失败导致数据丢失
