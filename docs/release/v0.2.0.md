# OfferFlow v0.2.0 Release Notes

发布日期：2026-06-18
节点说明：One-Shot Opportunity Radar / 一次分析 · 机会雷达版

## 概述

v0.2.0 在 v0.1.0 本地优先、Manual Mode 闭环之上，新增「一次复制粘贴即得机会雷达」的 One-Shot 工作流。

用户只需：在工具内填写岗位、公司与机会补充信息 → 复制**一次** One-Shot Prompt 给外部 AI → 把 AI 返回的完整内容（Markdown 报告 + `OFFER_FLOW_JSON` 数据块）**粘贴回工具一次**。工具自动解析出综合匹配度、公司画像、机会分、6 维机会雷达、风险等级、投递建议、面试关注点与 Boss 打招呼话术。

> v0.2.0 仍是 Manual One-Shot Mode：**不接 API、不做 BYOK、不做后端、不爬 Boss、不联网查公司**。它只解析你粘贴回来的固定 JSON，绝不自动调用 AI。

## 核心能力

1. **公司与机会补充表单**：公司规模 / 人员规模原文 / 公司类型 / 融资阶段 / 通勤时间 / 通勤方式 / 公司备注 / 机会备注，随岗位保存、回显，旧岗位自动补默认值兼容。
2. **One-Shot Prompt**：把简历/偏好、岗位基础信息、公司补充一次性拼接，要求外部 AI 输出 Markdown 报告 + 固定 `---OFFER_FLOW_JSON_START--- ... ---OFFER_FLOW_JSON_END---` 数据块（明确要求：未知标低置信度不乱猜、分数 0-100 整数、JSON 不含注释、枚举用固定英文代码、结构稳定）。
3. **OFFER_FLOW_JSON 解析器**：优先解析标记之间内容，兜底解析最后一个 ```json 代码块；归一分数到 0-100、枚举非法归 unknown、未给 opportunityScore 时按 6 维加权计算；返回 success / not_found / invalid_json / partial 状态与 warnings；永不抛未捕获异常。
4. **保存即自动解析回填**：保存 AI 原文时自动解析并写入 `companyAssessment` / `opportunityAnalysis` / `matchScore` / `report.greetingMessage`；原文永远优先落盘，解析失败 / 未找到 / 部分缺失均不阻断保存、不清空已有结构化字段，`bossGreeting` 为空时不覆盖已有话术。
5. **机会雷达卡**：主战场展示机会分（含等级）、综合匹配度、公司画像标签（规模/类型/稳定性/成长性/置信度）、风险等级、投递建议、决策摘要、面试关注点，以及自研 SVG 6 维机会雷达图；无数据显示空状态。
6. **列表升级与筛选排序**：新增公司规模、机会分列；按城市 / 公司规模 / 沟通状态 / 机会分下限筛选，按更新时间 / 机会分 / 匹配度排序；旧岗位无机会分显示 `-`，公司规模从 `companyInput` 兜底。筛选排序仅影响前端展示，不改持久化。
7. **浅色高级科技感视觉收口**：引入 Naive UI 作为唯一 UI 组件库，并把视觉语言折入真实页面 —— 顶部品牌区精致化、列表页为「机会资产池」卡片（非后台表格）、主战场为「单岗位机会简报」（卡片化分区 + 机会雷达主视觉）、表单卡片化与聚焦态，整体走类 Linear / Vercel / 飞书多维表格的清爽感。

## 数据模型增量（DEC-016）

在既有 `JobRecord` 上新增三字段，**不新增独立实体**：

- `companyInput: CompanyInput`（用户填写的公司与机会补充）
- `companyAssessment: CompanyAssessment | null`（AI 解析的公司画像）
- `opportunityAnalysis: OpportunityAnalysis | null`（AI 解析的机会分 / 6 维雷达 / 风险 / 建议 / 面试关注点 / 话术）

新增枚举：`CompanySizeTier` / `StabilityLevel` / `GrowthPotential` / `RiskLevel`。旧岗位读取时自动补默认值，不报错。

## 关键决策

- **DEC-016**：允许解析固定 `OFFER_FLOW_JSON` 数据块（仍不接 API / 不做 BYOK / 不做后端）。
- **DEC-017**：允许引入 Naive UI 作为唯一 UI 组件库（不引入图标库 / 路由 / 状态管理 / 图表库；雷达图自研 SVG）。

## 技术栈

- Vue 3 + Vite + TypeScript（strict）
- UI：Naive UI（唯一组件库）；机会雷达为自研 SVG 组件
- 本地存储：localStorage（driver 接缝，另含 memory driver 供自测）
- 无后端、无 AI API、无 BYOK；新增运行时依赖仅 `naive-ui`

## 质量基线

- `npm run typecheck`（vue-tsc --noEmit，strict）：0 error
- `npm run build`：成功
- `npm run selftest`：存储层 46/46 + OFFER_FLOW_JSON 解析器 43/43 通过
- 关键交互路径经真实浏览器自测（Vite dev + DOM 断言）：公司补充表单、保存即解析（标准 / 无 JSON / 非法 JSON / partial / 空话术）、机会雷达卡、列表筛选排序

## 已知限制

1. **一键复制**：在自动化预览沙箱中 `clipboard-write` 权限被置为 denied，无法在该环境完成实际写入验证；代码路径正确（安全上下文 + Clipboard API + execCommand 回退），真实 localhost 浏览器以真实点击应可正常复制，建议手动复核一次。
2. 机会雷达 / 公司画像依赖外部 AI 按约定输出 `OFFER_FLOW_JSON`；若外部 AI 未给该数据块，则只保存原文，结构化字段保持空（可重新粘贴或在表单手动维护匹配度）。
3. 视觉已收口到「浅色高级科技感」：列表、主战场、表单、外壳均已折入方案 B 风格；后续如需更精细的设计打磨（动效、暗色模式等）可另行评估，不在 v0.2.0 范围。
4. 保存未捕获 localStorage 配额溢出异常（与 Step 0 同源风险），超长文本极端情况下需关注。
5. 多标签页并发编辑同一岗位时，后保存者覆盖前者（单人本地场景可接受）。
