# OfferFlow v0.3.0 Release Notes

发布日期：2026-06-23

## 版本定位

v0.3.0 是 **半自动求职跟进决策台**。

它的主线是：

```txt
策略建议 + 下一步动作 + 推荐话术 + 止损判断
```

它不是：

```txt
CRM
自动投递工具
Boss 自动化工具
AI Chat
后端 SaaS
```

## 新增能力

1. **8 态沟通状态**
   - 使用 `communicationStatus` 替换旧 `contactStatus`。
   - 状态覆盖：`not_contacted`、`greeted_unread`、`greeted_read_no_reply`、`replied`、`interviewing`、`paused`、`closed`、`rejected`。

2. **跟进事实字段**
   - 新增 `lastGreetedAt`、`followupCount`、`lastFollowupAt`、`lastCommunicationNote`、`highValueSignal`、`strategyOverride`、`draftMessageText`。
   - 这些字段是用户手动维护的事实，可落库。

3. **只存事实，不存决策**
   - `strategy`、`nextAction`、`stopLoss`、`scenario`、`companyWarning` 均由规则实时派生。
   - 不持久化派生结果，不写入 `JobRecord` 或 localStorage。

4. **决策纯函数**
   - `deriveDecision(record, allJobs?)` 纯函数派生：
     - `strategy`
     - `nextAction`
     - `stopLoss`
     - `scenario`
     - `companyWarning`
   - `FOLLOWUP_COOLDOWN_DAYS = 3`，`MAX_FOLLOWUPS = 2`。

5. **详情页跟进决策面板**
   - 展示当前岗位的策略建议、下一步动作、止损状态、推荐话术场景和公司级预警。
   - 允许维护沟通状态、跟进次数、最近打招呼时间、最近跟进时间、高价值信号、沟通备注和话术草稿。

6. **推荐话术模板与复制**
   - 覆盖 `first_greeting`、`second_followup`、`final_unread_followup`、`high_salary_low_match_probe`、`premium_but_cold_closing`、`hr_reply_bridge`。
   - 支持变量安全兜底、复制推荐话术、填入草稿、编辑并保存 `draftMessageText`。

7. **列表页决策台模式**
   - 列表行展示 `strategy`、`nextAction`、`stopLoss`。
   - 新增待打招呼、可跟进、待止损、等回复筛选 chips。
   - 新增按决策优先级排序。

8. **公司级只读预警**
   - 基于现有 `JobRecord[]` 派生同公司岗位预警。
   - 不新增 Company 实体，不新增公司状态，不写入公司聚合数据。

## 关键决策

- DEC-020：`communicationStatus` 替换 `contactStatus`，并记录旧状态迁移映射。
- DEC-021：只持久化跟进事实字段，不持久化派生决策。
- DEC-022：决策结果由 `deriveDecision` 纯函数实时派生，止损阈值常量化为 `FOLLOWUP_COOLDOWN_DAYS = 3`、`MAX_FOLLOWUPS = 2`。
- DEC-024：公司级预警只从 `JobRecord[]` 派生，不新增 Company 实体。

## 边界

v0.3.0 不做：

1. 不接 API
2. 不做 BYOK
3. 不做后端
4. 不爬 Boss
5. 不自动检测 Boss 已读 / 未读
6. 不自动发送消息
7. 不自动投递
8. 不做定时发送
9. 不做模拟点击
10. 不做提醒系统
11. 不做完整 AI Chat
12. 不做完整沟通日志
13. 不做 append-only message log
14. 不做批量操作
15. 不做联系人管理
16. 不做多渠道聚合
17. 不做漏斗转化率大看板
18. 不做用户自定义规则引擎
19. 不做 CRM
20. 不新增 Company / Contact / Message / JobStatusLog / FollowupLog / Reminder 实体
21. 不持久化 `strategy` / `nextAction` / `stopLoss` / `scenario` / `companyWarning`

## 升级说明

- 旧 `contactStatus` 会迁移为 `communicationStatus`。
- 旧 `contactStatusUpdatedAt` 不迁移为 `lastGreetedAt`，避免把“任意沟通状态更新时间”伪装成“最近一次打招呼时间”。
- 旧记录缺少 `lastGreetedAt` / `lastFollowupAt` 时不会崩；冷却期判断会保守等待，直到用户维护可信时间事实。
