# OfferFlow v0.1.0 Release Notes

发布日期：2026-06-15
节点说明：P0 manual-mode closed loop completed（P0 手动模式闭环完成）

## 概述

v0.1.0 是 OfferFlow 的首个封版节点。完成了本地优先、Manual Mode、不接任何 AI API 的完整求职台账闭环（Step 0 – Step 8）。

核心闭环：

> 简历/偏好配置 → 录入岗位 JD → 生成分析 Prompt → 复制到外部 AI → 把结果粘贴回岗位 → 沉淀报告原文 / 编辑 Boss 话术 → 维护沟通状态 → 面试前从列表回看。

## 核心能力

1. 简历 / 偏好全局配置（9 字段，覆盖式保存，刷新回显）
2. 岗位台账列表（空状态、列表展示、新建入口、点击进入详情）
3. 岗位主战场基础信息录入与保存（新建创建 / 编辑更新闭环）
4. 结构化分析 Prompt 生成 + 一键复制（含外部 AI 操作提示，无 `{{}}` 残留）
5. 外部 AI 结果原文兜底承接（原文必存、粘贴时间、parseStatus 标记，任意格式不阻断）
6. 分析报告原文展示 + Boss 打招呼话术编辑 / 保存 / 复制
7. 沟通状态流转（6 态：未沟通 / 已打招呼 / 已回复 / 已约面 / 已拒绝 / 已结束，即时持久化、列表同步）
8. 列表 ↔ 详情回看闭环，全字段回显，面试前可回溯报告与话术
9. 全本地存储（localStorage），刷新不丢，无后端、无登录

## 技术栈

- Vue 3 + Vite + TypeScript（strict）
- 本地存储：localStorage（driver 接缝，另含 memory driver 供自测）
- 无后端、无 AI API、无新增运行时依赖（仅 `vue`）

## 质量基线

- `npm run typecheck`（vue-tsc --noEmit，strict）：0 error
- `npm run build`：成功
- `npm run selftest`：30/30 通过（存储层 CRUD 全覆盖）
- 关键交互路径经真实浏览器自测（Vite dev + DOM 断言）

## 已知限制

1. **一键复制**：在自动化预览沙箱中 `clipboard-write` 权限被置为 denied，无法在该环境完成实际写入验证；代码路径正确（安全上下文 + Clipboard API + execCommand 回退），真实 localhost 浏览器以真实点击应可正常复制，建议手动复核一次。
2. v0.1 **不自动结构化解析** AI 报告：报告以原文兜底展示，仅 Boss 话术可手动编辑；jobType / keywords / applyAdvice 等结构化字段暂不在 UI 编辑。
3. `matchScore` 在列表展示但主战场未提供编辑入口。
4. 沟通状态切换刷新 `contactStatusUpdatedAt`，但不改 `updatedAt`，故不因状态切换在列表顶置。
5. 保存未捕获 localStorage 配额溢出异常（与 Step 0 同源风险），超长文本极端情况下需关注。
6. 多标签页并发编辑同一岗位时，后保存者覆盖前者（单人本地场景可接受）。

## 数据模型（未变更核心字段）

- 全局配置 `JobSeekerProfile`（单条，覆盖式）
- 岗位记录 `JobRecord`（多条，一条一个 storage key）：基础信息 + `promptText` + `aiRawResult` / `aiPastedAt` / `parseStatus` + `report`（`JobReport`，默认 null）+ `matchScore` + `contactStatus` / `contactStatusUpdatedAt`
- 未引入 PromptRecord / AIAnalysisResult / JobStatusLog 独立实体（DEC-004）

## 下一阶段 P1 方向（未实现）

- Prompt 模板配置（查看 / 编辑 / 恢复默认）
- 简单统计（岗位总数 / 各状态数量）

> P1 是否启动、启动顺序由用户另行决定；Personal BYOK 未纳入 v0.1.0，后续如需讨论必须另行拍板。
